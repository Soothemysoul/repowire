# beads-nfap.1 — отчёт по метрике токенов (out-of-band ACK-receipts)

**Дата:** 2026-06-20 · **Автор:** backend-worker · **Repo:** repowire-fork
**Артефакт DoD (б):** замер экономии контекста до/после на типовой mesh-сессии.

---

## Что измеряем

До изменения каждый `notify_peer`/`ask_peer`/`broadcast` порождал у ОТПРАВИТЕЛЯ
отдельный conversation-turn с распиской о доставке (AUTO-ACK), а делегирование —
ещё и intent-ACK. Харнесс поверх каждого инжекта вешает system-reminder «MUST
address the user's message». Эти токены тратятся на сообщения, которые модель по
правилу (3 do-not-reply триггера в `peer-communication.md`) и так обязана
игнорировать.

После изменения (кандидат B, hooks-only) расписки перехватываются на стороне
отправителя и пишутся в per-pane ack-state-файл — **в контекст модели не
инжектится ничего**. До сессии доходит только actionable-сигнал (AUTO-NACK или
эскалация watchdog при таймауте) — одним инжектом.

## Методика

Скрипт: `scripts/measure_oob_receipt_savings.py` (воспроизводимо,
`uv run python scripts/measure_oob_receipt_savings.py`).

Оценка токенов — эвристика `chars/4` (tiktoken в runtime-окружении недоступен).
Это **оценка**: абсолютные числа приближают BPE-токенизатор, но соотношение
(after = 0) точное при любом токенизаторе. Замерялись точные строки payload'ов,
как они ложатся в pane отправителя (`websocket_hook` инжектит notify как
`@{from_peer}: {text}`), плюс наблюдённый в этой же сессии текст
harness-reminder'а.

Стоимость одной расписки (payload + harness-reminder):
- AUTO-ACK: **~78 ток**
- intent-ACK: **~65 ток**

## Результат (до / после)

| Профиль сессии | notify | до (ток) | после (ток) |
|---|---:|---:|---:|
| лёгкая (10 notify, 30% хэндоффов) | 10 | 975 | **0** |
| типовая (30 notify, 50% хэндоффов) | 30 | 3315 | **0** |
| тяжёлая (60 notify, 50% хэндоффов) | 60 | 6630 | **0** |
| цепочка делегирования (15, все хэндоффы) | 15 | 2145 | **0** |

**Типовая mesh-сессия: ~3300 токенов шума на отправителя → 0.** В каждой
параллельной сессии mesh'а экономия отдельная (расписки засоряли контекст у
каждого отправителя независимо).

## Что НЕ глушится (инвариант сохранён)

- **AUTO-NACK** (реальный провал доставки) — по-прежнему инжектится (actionable).
- **Watchdog-эскалация** — если за 60с нет ACK, отправитель получает ОДИН
  actionable-инжект (не тишина).
- Интерактивный query-response (`/response`) не затронут — это не receipt.
- `REPOWIRE_RECEIPT_INLINE=1` мгновенно возвращает inline-инъекцию (rollback).

## Оговорки

- `chars/4` занижает/завышает реальный BPE-счёт в пределах ~±15%; порядок
  величины и after=0 — устойчивы.
- Профили «notify/сессия» — оценочные; реальная экономия линейна по числу
  исходящих notify, так что для интенсивных оркестраторов (director/pm) она выше.
